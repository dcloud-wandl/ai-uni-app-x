# 代码生成

这是一个uni-app x初始化工程模板，需要兼容iOS、安卓、鸿蒙、H5和微信小程序，请基于此工程实现一个任务管理系统。

## 任务管理系统需求描述

### 任务列表页（App首页）：

显示任务列表，列表项中显示任务标题、优先级、截止日期；点击可跳转任务详情页；

任务列表，支持左滑出操作菜单，操作菜单有两个：编辑、删除，点击编辑打开任务编辑页，删除直接删除该任务。

空状态：当没有任务时，显示一个居中的“暂无任务”提示。

页面底部显示一个Fab悬浮按钮，点击可打开新增任务的页面。Fab 按钮需要固定在屏幕右下角，且不随列表滚动。

### 任务详情页 ：

包含：任务标题、任务描述、截止日期、优先级（分为高/中/低）。

任务详情支持编辑，编辑完毕后自动刷新任务详情页。从任务详情页返回任务列表页，任务列表页会自动刷新。

任务详情页支持删除，删除后返回任务列表页，并自动刷新任务列表。

### 新增任务页：

新增任务页支持录入任务标题、任务描述、截止日期、优先级（分为高/中/低）。

支持表单校验：标题不能为空，截止日期不能早于今天。

任务新增成功后，返回任务列表页，并自动刷新任务列表。


### 数据持久化：

要求数据做本地存储，应用重启后数据不丢失。


### UI风格

主体 UI 要求使用Vue绿。

## 工作需求

自动探测编译日志，解决编译错误；
自动探测运行日志，若有运行错误，自动解决；

# 自动化测试脚本生成


对项目中的每个页面生成自动化测试脚本

应用启动后进入首页，点击右下角的fab按钮，打开新增任务页面，输入第一个任务。

第一个任务的信息如下：
```
任务名称：测试任务-01
描述：任务描述 001
截止日期：明天
优先级：低
```

任务创建成功后自动返回首页，也就是任务列表页，自动化测试脚本要断言当前任务列表页有一条记录。

点击第一条任务，打开任务详情，在任务详情页点击编辑，打开任务的编辑页面，将该任务内容修改为：

```
任务名称：测试任务-01-v2
描述：任务描述 001-v2
截止日期：明天
优先级：中
```

任务编辑后自动返回任务详情页，并且任务详情页内容自动刷新，需要断言判断是否生效。

任务详情页返回任务列表页，任务列表页也会刷新，需要通过断言判断是否生效。


任务列表页，点击右下角的fab按钮，打开新增任务页面，输入第2个任务。

第2个任务的信息如下：
```
任务名称：测试任务-02
描述：任务描述 002
截止日期：后天
优先级：中
```

任务创建成功后自动返回任务列表页，自动化测试脚本要断言当前任务列表页有2条记录。

向左滑动第2条任务，露出编辑和删除按钮，点击编辑按钮，打开第2条任务的编辑页，将第二条任务内容修改如下：

```
任务名称：测试任务-02-v2
描述：任务描述 002-v2
截止日期：5天后
优先级：高
```

任务编辑后自动返回任务详情页，并且任务详情页内容自动刷新，需要断言判断是否生效。

任务详情页返回任务列表页，任务列表页也会刷新，需要通过断言判断是否生效。


任务列表页，点击右下角的fab按钮，打开新增任务页面，输入第3个任务。

第3个任务的信息如下：
```
任务名称：测试任务-03
描述：任务描述 003
截止日期：后天
优先级：中
```

任务创建成功后自动返回任务列表页，自动化测试脚本要断言当前任务列表页有3条记录。

向左滑动第3条任务，露出编辑和删除按钮，点击删除按钮，如果有删除确认框，要确认删除，然后断言任务列表剩余 2 条记录。
